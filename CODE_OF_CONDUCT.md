# Code of Conduct

## Our Commitment

FrontierWarden is a trust and reputation infrastructure project for EVE Frontier, built on the Sui blockchain. The integrity of this system — on-chain and off — depends on the same principles we encode in the protocol: transparency, accountability, and proof-backed behavior.

We are committed to making participation in FrontierWarden's development a respectful and productive experience for everyone, regardless of background, experience level, or affiliation.

## Standards

### Expected Behavior

- Communicate clearly and with evidence. Vague claims waste everyone's time.
- Respect that tribe operators, tool builders, and core contributors have different contexts and incentives.
- Raise concerns about protocol design, trust model, or API behavior with specifics — schema IDs, transaction digests, endpoint responses, or reproducible steps.
- Treat security disclosures with discretion. See [SECURITY.md](SECURITY.md) for the private disclosure process.
- When reviewing code or ADRs, critique the design decision, not the person who made it.
- Accept that trust model decisions (tenant policy, attestation scope, gate authority) are intentionally opaque to outside parties — that is by design, not an oversight.

### Unacceptable Behavior

- Harassment, intimidation, or personal attacks in issues, PRs, or Discussions.
- Unauthorized scraping, extraction, or reverse-engineering of reputation data or private tenant policy state.
- Submitting deliberately misleading attestations, vouches, or test data that pollutes on-chain state.
- Public disclosure of security vulnerabilities before coordinated private disclosure.
- Impersonating Kodaxadev, FrontierWarden infrastructure, or oracle identities in any context.
- Attempting to use this project or its tooling to build systems that harm, surveil, or deceive EVE Frontier players without their informed consent.

## Scope

This Code of Conduct applies to all project spaces — GitHub Issues, Pull Requests, Discussions, the Wiki, and any official communication channel linked from this repository.

It also applies when representing FrontierWarden in external communities (EVE Frontier Discord, Sui developer forums, or other game/blockchain forums).

## Enforcement

Violations can be reported privately to the maintainer:

**Justin Davis — Justin.DavisWE@icloud.com**

All reports will be reviewed with confidentiality. The maintainer will respond within a reasonable time with a determination and, if warranted, a corrective action. Corrective actions may include a warning, temporary restriction, or permanent removal from project spaces.

Acting in bad faith when reporting (false reports intended to harm a contributor) is itself a violation of this Code of Conduct.

## Attribution

This Code of Conduct is written for the specific context of FrontierWarden and reflects the trust-first values embedded in the protocol itself.
